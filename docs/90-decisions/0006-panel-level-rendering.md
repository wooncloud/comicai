# ADR-0006: 패널 단위 렌더링 (페이지 단위 미사용)

- 상태: Accepted
- 날짜: 2026-05-16

## 컨텍스트
- 만화 한 페이지는 여러 패널(컷)로 구성.
- 페이지 통째 렌더 vs 패널 단위 렌더의 트레이드오프 존재.
- 현재 Gemini/GPT는 만화 페이지 레이아웃 통제력이 약함.

## 결정
**패널 단위 렌더링**을 채택. 사용자가 패널 도형을 그린 후, 각 패널을 독립 잡으로 렌더한다.

## 대안
1. **페이지 통째 렌더** — 컷 구도 일관성은 좋을 수 있으나 모델이 패널 분할을 정확히 따라가기 어려움. 재렌더 비용 큼.
2. **하이브리드(페이지 1차 → 패널 보정)** — 복잡도 큼. MVP에 부적합.
3. **패널 단위** — 일관성 정보 멘션을 패널별로 정교 제어 가능. 실패 시 해당 패널만 영향. **채택**.

## 결과
**긍정**:
- 패널별 독립 잡 → 일부 실패해도 다른 패널 무관.
- 패널 단위 멘션·콘티·텍스트 정밀 제어.
- 재렌더 단위가 작음 → 사용자 비용 절감.

**부정**:
- 페이지 전체의 컷 간 시각 일관성(빛, 시간대 등)은 부족할 수 있음.
- 향후 페이지 모드 추가 시 별도 설계.
