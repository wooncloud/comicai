## 작성 원칙

- 모든 주장은 `file:line` 인용을 동반한다.
- 코드와 어긋나면 코드가 진실이다. 문서를 갱신한다.
- 아직 구현되지 않은 계획은 `develop-docs/` 또는 PR 본문에 둔다.
