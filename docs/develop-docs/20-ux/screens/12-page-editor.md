# 화면: 페이지 에디터 ⭐

> v0.1 — 2026-05-16 — Draft
> 본 화면이 제품 가치의 ~90%. 와이어프레임은 상태별 다중.

## 1. 목적

한 페이지에 대해 패널 도형 그리기 + 패널별 콘티/텍스트/이미지/멘션 입력 + 모델 선택 + 렌더링 + 결과 확인.

## 2. 진입 / 이탈

- 진입: 페이지 목록 카드 클릭 또는 프로젝트 홈에서.
- 이탈: 뒤로(프로젝트 홈), 다른 페이지로 이동.
- 관련 기능: E-03~E-12.

## 3. 상태

- 패널 미선택
- 패널 선택 + 입력 중
- @멘션 자동완성 열림
- 렌더링 진행 중
- 렌더링 성공
- 렌더링 실패
- 히스토리 트레이 열림
- 내보내기 모달 ([`13-page-export.md`](./13-page-export.md))

## 7. 인터랙션

### 도구

- `V` 선택 / `P` 패널 / `D` 콘티 / `T` 텍스트 / `🖼` 이미지 업로드.
- 패널 그리기: 드래그로 사각형/자유 도형. 테두리 색/굵기/투명 선택 가능.
- 콘티: 펜/지우개. 패널 선택 후에만 활성.

### 키 단축어

[`../01-interaction-principles.md`](../01-interaction-principles.md) §6 참조. `Enter`로 렌더링.

### 자동 저장

1.5s 디바운스. 저장 실패 시 Topbar 빨간 배너.

### 렌더링

- "렌더링" 버튼 → 잡 enqueue → SSE 구독 → 상태 변화.
- 다른 패널 선택/편집은 계속 가능 (비차단).

## 8. 데이터 의존성

| 데이터                      | 출처                                   |
| --------------------------- | -------------------------------------- |
| 페이지 메타                 | `GET /v1/pages/:id`                    |
| 패널 목록                   | `GET /v1/pages/:id/panels`             |
| 패널 갱신 (자동저장)        | `PATCH /v1/panels/:id`                 |
| 일관성 정보 (멘션 자동완성) | `GET /v1/projects/:pid/consistency`    |
| 렌더 요청                   | `POST /v1/panels/:id/render`           |
| 렌더 진행                   | `GET /v1/render-jobs/:id/events` (SSE) |
| 히스토리                    | `GET /v1/panels/:id/history`           |

## 9. 엣지 케이스

- 입력 비어있는 채로 렌더 요청 → 버튼 비활성화 + 툴팁.
- 다른 탭에서 같은 페이지 열림 → 새 탭은 read-only 잠금 안내.
- 캔버스에서 패널이 페이지 밖으로 나갈 시 → 자동 클램프.
- 패널이 너무 작음(< 64px) → 렌더 거부 + 안내.
- 멘션된 엔티티가 삭제됨 → 노드 회색 + ⚠ + 렌더 시도 시 차단.
- 네트워크 끊김: SSE 자동 재연결 + 잡 상태 폴링 fallback.
