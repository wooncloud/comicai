# ADR-0001: MVP에 BYOK(Bring Your Own Key) 채택

- 상태: Accepted
- 날짜: 2026-05-16

## 컨텍스트

- 사업 단계가 아님. 1인 개발, 자체 인프라(개인 PC).
- AI 모델 호출(Gemini, OpenAI)은 비용 발생 → MVP에서 운영자가 비용 부담은 비현실적.
- 결제·크레딧 시스템 구축은 큰 작업이며 핵심 가치 검증에 직접 기여하지 않음.

## 결정

MVP는 **BYOK 방식**으로 출시. 사용자가 직접 Gemini / OpenAI API 키를 발급·등록해 사용한다. 크레딧·결제 인프라는 데이터 모델로만 준비하고 실제 결제 PG는 연동하지 않는다.

## 대안

1. **운영자가 API 비용 부담 + 크레딧 판매** — MVP 단계에 비용·결제 PG 부담이 큼.
2. **체험판 + 결제** — 결제 시스템 필요, 동일 부담.
3. **현재 결정(BYOK)** — 채택.

## 결과

**긍정**:

- 운영 비용 0, 결제 시스템 미연동.
- 핵심 가치(일관성)에 집중해 검증 가능.
- 사용자가 자기 API 한도 안에서 자유롭게 시험.

**부정**:

- 진입 장벽: API 키 발급 절차 학습 필요 → 타깃이 기술 친화 얼리어답터로 좁아짐.
- 보안 부담: 사용자 키 보관 책임 → AES-256-GCM 암호화, 마스킹 미들웨어 필수.
- 향후 크레딧제로 전환 시 사용자 흐름 변경 비용.
