# 인수 기준 (Acceptance Criteria)

> v0.1 — 2026-05-16 — Draft
> 기능 ID별 Given/When/Then. 모든 기능을 다 채우는 게 아니라, 모호한 기능 위주.

## E-06 — 텍스트 입력 + @멘션 자동완성
- **Given** 프로젝트에 캐릭터 "주연"이 등록되어 있고
- **When** 사용자가 패널 텍스트 입력란에 `@주` 까지 입력하면
- **Then** 드롭다운에 "주연(@char_…)"이 표시되고
- **When** Enter 또는 클릭으로 선택하면
- **Then** 본문에 멘션 노드가 삽입되고(텍스트 raw가 아닌 노드), 라벨 "주연"이 표시된다.

## E-08 — 렌더링 실행
- **Given** 패널에 텍스트 또는 콘티가 입력되어 있고 API 키가 등록되어 있을 때
- **When** "렌더링" 버튼 클릭
- **Then** 패널 상태가 "QUEUED" → "RUNNING" → "SUCCEEDED"로 전이되며
- **And** 결과 이미지가 패널 영역에 anchor되어 표시된다
- **Edge**: 입력이 비었으면 버튼 비활성화 + 툴팁 "텍스트나 콘티가 필요합니다".

## C-06 — 일관성 정보 수정 시 과거 렌더 보존
- **Given** 캐릭터 "주연" v2로 렌더된 패널이 존재할 때
- **When** "주연"을 수정해 v3가 되어도
- **Then** 과거 RenderJob의 IR 스냅샷에는 v2 데이터가 그대로 유지된다
- **And** 새 렌더는 v3 기준으로 동작한다.

## U-05 — API 키 암호화 저장
- **Given** 사용자가 Gemini API 키 입력
- **When** 저장 버튼 클릭
- **Then** DB에는 AES-256-GCM 암호화된 ciphertext만 저장된다
- **And** 어떤 로그(앱/DB/Nest/Next/Docker)에도 평문 키가 출현하지 않는다 (검증: 정규식 grep).

## E-10 — 히스토리 FIFO
- **Given** 한 패널에 렌더가 20회 누적되어 있고
- **When** 21번째 렌더가 성공하면
- **Then** 가장 오래된 1번째 결과가 큐에서 제거된다
- **And** 사용자는 21번째 결과를 현재로 보고 있다.

> 추가 기능은 개발 진행 중 보강.
