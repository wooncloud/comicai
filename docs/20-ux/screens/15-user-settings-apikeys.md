# 화면: 설정 — API 키 관리 (BYOK)

> v0.1 — 2026-05-16 — Draft
> MVP의 핵심 화면. 사용자가 자신의 Gemini/OpenAI 키를 등록·관리.

## 1. 목적
사용자가 모델사 API 키를 등록·검증·삭제. MVP는 BYOK이므로 키 없으면 렌더 불가.

## 2. 진입 / 이탈
- 진입: 가입 직후 안내, 설정 메뉴, 렌더 실패 시 에러 카드 링크.
- 관련 기능: U-04, U-05.

## 3. 상태
- 빈 (가입 직후)
- 정상 (등록된 키 목록)
- 검증 중
- 검증 실패

## 4. 와이어프레임

### 빈 상태
```
┌───────────────────────────────────────────────────────┐
│ 설정 / API 키                                          │

│  · Gemini (Google AI Studio에서 발급)                 │
```

## 5. 컴포넌트
shadcn `Card`, `Dialog`, `Input` (type=password 토글 가능), `Button`, `Badge`.

## 6. 인터랙션
- "검증 후 저장": 모델사에 테스트 호출 → 성공 시 암호화 저장.
- 키 표시는 항상 마스킹. 사용자도 평문 다시 못 봄(검증만).
- "재활성": 회로 차단 해제 + 즉시 재검증.

## 7. 데이터 의존성
- `GET/POST/DELETE /v1/api-keys`.
- `POST /v1/api-keys/:id/verify`.

## 8. 엣지 케이스
- 키 형식 오류 → 클라이언트 사전 검증 (예: Gemini는 `AIza...`).
- 검증 호출 자체 실패 → "네트워크 오류, 잠시 후 재시도".
- 같은 provider 키 중복 등록 → 기존 교체 확인 모달.
