# 화면: OAuth 콜백

> v0.1 — 2026-05-16 — Draft

## 1. 목적
OAuth 제공자에서 리다이렉트된 사용자를 처리하고 세션 생성.

## 2. 진입 / 이탈
- 진입: `/auth/:provider/callback?code=...`.
- 이탈: 성공 → `/dashboard`. 실패 → `/login?error=...`.

## 3. 상태
- 처리 중 (스피너) / 성공 / 실패.

## 4. 와이어프레임

```
┌────────────────────────────────┐
│                                │
│        로그인 처리 중...        │
│             ◌                  │
│                                │
└────────────────────────────────┘
```

## 5. 컴포넌트
스피너 + 텍스트만.

## 6. 인터랙션
- 자동 처리. 사용자 입력 없음.

## 7. 데이터 의존성
- `GET /v1/auth/oauth/:provider/callback`.

## 8. 엣지 케이스
- 토큰 교환 실패 → 에러 페이지.
- 이메일 중복 → 계정 연결 모달.
